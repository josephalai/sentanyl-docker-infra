### BUG-014: handleAddUserToStory missing FireOutboundWebhook call
- **Found**: #18 outbound-webhooks: StoryStarted webhook never delivered
- **Root cause**: The POST `/api/user/:userId/story/:storyId` endpoint (`handleAddUserToStory`) didn't call `FireOutboundWebhook`. Only `handleStartStory` (PUT route) and `AddBadgeToUser` had the call.
- **Fix**: Added `FireOutboundWebhook(WhStoryStarted)` to `handleAddUserToStory` in serve_handlers.go

### BUG-015: handleEmailClicked user lookup fails when subscriber_id != creator public_id
- **Found**: #18 outbound-webhooks: Click returned 200 but no trigger matched
- **Root cause**: `handleEmailClicked` passed `e.SubscriberId` (= creator's public_id) to `GetUserByEmailAddress`, which queries `{subscriber_id: X, email: Y}`. Users created via MongoDB have `subscriber_id` set to the account subscriber ID, not the creator's public_id. Mismatch → user not found → no triggers executed.
- **Fix**: Changed `handleEmailClicked` and `handleEmailOpened` to call `GetUserByEmailAddress("", e.EmailAddress)` — look up by email only. The hot_trigger query already scopes to the correct creator.
